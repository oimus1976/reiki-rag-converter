# Judgment

## プライベートナレッジにおける正本形式選択（Markdown）

---

## 1. Judgment 宣言（結論）

本 Judgment は、例規データを **プライベートナレッジとして格納・利用する場合の正本形式**について判断するものである。

**結論として、正本形式は Markdown（Markdown 互換テキスト）とする。**

---

## 2. Judgment Scope（判断の適用範囲）

本判断は、以下の条件下において適用される。

* 対象データ

  * 自治体例規（条例・規則等）の本文データ
* 利用目的

  * 個人または組織内での **プライベートナレッジ格納**
  * LLM / RAG による検索・再解釈・説明生成
* 前提条件

  * ナレッジ投入時に HTML の DOM 構造は保持されない
  * 検索・推論はテキストベースで行われる

本 Judgment は、
**Web 公開用データ形式の選択**や
**人間が直接閲覧するための表示形式の優劣**を扱わない。

---

## 3. Judgment に用いた観測枠組み

本判断は、HTML 形式と Markdown 形式の例規データを
**同一条件でプライベートナレッジに格納し、同一質問を投入した比較実験**に基づいて行われた。

観測は、以下の枠組みに基づいて整理された。

* **到達性（Retrieval Reach）**

  * 質問で要求された条・項・附則に正しく到達できているか
* **構造保持性（Structural Integrity）**

  * 条文が持つ論理構造（定義・本文・例外・附則等）が意味として保持されているか
* **補完耐性（Hallucination Control）**

  * 条例に記載されていない内容を、過剰に補完していないか

※ 本枠組みは評価軸ではなく、
**入力形式の違いによって生じる意味の揺れを観測するための整理単位**である。

---

## 4. 観測結果の要約（評価ではない）

上記枠組みに基づく観測から、以下の傾向が確認された。

* HTML 形式では、

  * 視覚的・DOM 的には構造を持っているが、
    ナレッジ投入後の前処理により構造情報が失われ、
    条文の意味解釈が文脈の偶然性に依存しやすい。
* Markdown 形式では、

  * 見出し、条番号、本文の順序が
    あらかじめ「意味の列」として表現されており、
    ナレッジ投入後も解釈の揺れが比較的小さい。

これらは **回答の巧拙や自然さの差**ではなく、
**入力形式が意味の再解釈に与える影響の差**として観測されたものである。

---

## 5. Judgment（判断）

以上の観測を踏まえ、

* プライベートナレッジ用途において重要なのは
  情報量や表示忠実性ではなく、
  **意味が安定して再解釈されること**である。
* Markdown 形式は、
  ナレッジ化および再解釈の過程で失われる前提を織り込んだ形式であり、
  意味の揺れが小さい。

よって、

**例規データをプライベートナレッジとして扱う場合、
Markdown（Markdown 互換テキスト）を正本形式として採用する。**

---

## 6. 非目的および留保事項

本 Judgment は、以下を意図しない。

* HTML 形式の有用性を否定すること
* 将来にわたる再評価を禁止すること
* モデル性能や回答品質の優劣を断定すること

本判断は、
**v0.2 時点の観測結果に基づく判断の固定**であり、
新たな前処理方式・検索方式・利用目的が導入された場合には、
別途再評価され得る。

---

## 7. Judgment の位置づけ

本 Judgment は、

* HTML vs Markdown 比較実験に対する
  **解釈の終端宣言**
* プライベートナレッジ設計における
  **正本形式選択の確定点**

として扱われる。

本 Judgment 以降、
同一条件下における形式選択の再解釈は行わない。
