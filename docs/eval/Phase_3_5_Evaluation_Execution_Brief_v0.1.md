# Phase 3.5 Evaluation Execution Brief（雛形）

本フェーズにおける「Execution」とは、  
評価対象に対する回答生成および観測ログ取得を指し、  
評価結果の確定や判断を含まない。

## 0. 本文書の位置づけ

本書は、
**Phase 3 Evaluation Brief v0.1 で確定した評価設計を前提として、
実際に評価を「実行」するための作業ブリーフ**である。

本フェーズ（Phase 3.5）は、

- 評価を *設計から実行へ* 移行する
- ただし **判断・結論は出さない**

という中間フェーズとして位置づけられる。

---

## 1. Phase 3.5 の目的

Phase 3.5 の目的は、
Phase 3 で定義された評価軸および評価シナリオに基づき、

- Markdown 単独利用
- Markdown＋CSV 併用

の回答を実際に生成・比較し、
**評価可能な観測ログを取得すること**である。

本フェーズでは、

- CSV 併用の是非を判断しない
- 正解・不正解を確定しない

---

## 2. 前提条件（Phase 3 からの継承）

- Phase 3 Evaluation Brief v0.1 は **design frozen**
- 評価軸・評価シナリオは **変更不可**
- CSV スキーマ・RAG 構成は **固定**
- 実行対象条例・質問セットは **明示的に限定**

---

## 3. 評価対象と比較構成

### 3.1 比較構成

- A：Markdown 単独
- B：Markdown＋CSV 併用

※ Phase 3 と同一構成
※ 実行条件は完全に揃える

---

### 3.2 評価対象条例・質問セット

- 対象条例：

  - （例）○件（理由を記述）
- 対象質問：

  - Phase 3 で定義されたシナリオに対応する質問のみ

※ 網羅性は目的としない

---

## 4. 実行手順（概要）

本フェーズにおける実行は、
以下の流れで行う。

1. 評価対象条例・質問の確定
2. A/B 各構成での回答生成
3. 回答結果の保存
4. 評価軸に沿った観測ログの記録

詳細手順は
Evaluation 実行手順 v0.1 を参照する。

---

## 5. 観測ログの構成

本フェーズで取得する観測ログは、
**評価軸ごとの変化を後続フェーズで判断可能にするための材料**である。

### 5.1 必須観測項目（例）

- 該当評価シナリオ ID
- 使用構成（A / B）
- 回答本文
- 条文参照有無
- 表（CSV）参照有無
- 主評価軸に対応する観測メモ

---

### 5.2 記述ルール

- 良否・結論を書かない
- 「正しい」「間違い」という語を使わない
- 事実・挙動・参照関係のみを記録する

観測ログは、後続フェーズで機械的・横断的に参照可能であることを前提とし、
自由記述のみで構成されることは想定しない。

---

## 6. 本フェーズで行わないこと（再確認）

- Judgment の記録
- Gate 判定
- スコアリング
- CSV / Markdown / RAG の改修
- モデル比較・最適化

---

## 7. 完了条件（Phase 3.5）

Phase 3.5 は、以下をもって完了とする。

- Phase 3 で定義された評価シナリオについて
  - A / B 両構成の回答が取得されている
  - 観測ログが欠損なく記録されている
- 後続フェーズ（Phase 4）が
  - 判断に進める状態であることが確認できる

なお、評価軸に基づく観測が成立しない場合は、  
Phase 3.5 を完了とせず、Phase 3 設計への差し戻しを検討する。

---

## 8. 次フェーズへの引き渡し

Phase 3.5 の成果は、

- 評価実行ログ
- 観測記録

として Phase 4（Evaluation / Judgment）に引き渡される。

本書自体は、
**最初の実行完了時点で status を fixed に更新**し、
以後は改訂対象としない。

---

## 9. Notes（未確定・運用上の留意）

- 実行回数は最小限とする
- 再実行が必要な場合は run_id を明示的に分ける
- Phase 3 設計との乖離が生じた場合は実行を中断する
