# Migration Guide  

## Evaluation HUMAN 記録 v0.1 → v0.2

---

## 目的

本ドキュメントは、**Evaluation HUMAN 記録 v0.1** を  
**v0.2 の設計思想・責務分離に沿って移行するための公式ガイド**である。

v0.2 では以下を目的とする。

- AUTO / HUMAN の責務分離を明確化する
- HUMAN 記録を「判断補助ログ」に限定する
- 将来の自動集約・比較・再利用に耐える構造を確保する

---

## 1. v0.1 と v0.2 の根本的な違い

### v0.1（旧）

- HUMAN が以下を混在して記録していた
  - 差分の事実要約
  - 所感・気づき
  - 評価・判断に近い表現
- AUTO 出力との関係が暗黙
- run 単位の再現性が弱い

### v0.2（新）

- HUMAN 記録は **AUTO JSON を前提としてのみ存在**
- HUMAN は以下を記載しない
  - 事実の再集計
  - 差分の再解釈
  - 良否・合否・評価・結論
- HUMAN 記録の役割は以下に限定される  
  **「なぜ今は判断できないか」  
  「次に何を比較・確認すべきか」**

---

## 2. 移行の基本原則（必読）

### 原則 1：AUTO が書いた事実は HUMAN に書かない

以下は **evaluation_auto.json の責務**であり、  
HUMAN 記録には記載しない。

- diff 件数
- true / false の分布
- 条例数・質問数の集計

---

### 原則 2：v0.1 の「所感」はそのまま移さない

以下の表現は v0.2 では禁止または再構成対象とする。

| v0.1 にありがちな表現 | v0.2 での扱い |
|---|---|
| 想定どおりだった | 削除 |
| 問題がある | 削除 |
| 良い傾向 | 削除 |
| 気になる点 | 「注目した差分」として再記述 |

---

### 原則 3：v0.2 では「判断しない」ことを書く

v0.2 HUMAN 記録の主語は常に次である。

> なぜ、この時点では判断できないのか  
> 次にどの観点を確認する必要があるのか

---

## 3. セクション別 移行ガイド

### 3.1 v0.1 Summary / 概要

**v0.1**
```md
Reference Diff が一部 false であり、全体として問題はなさそう。
````

**v0.2**

* 削除

※ Summary は AUTO JSON に完全移管されている。

---

### 3.2 v0.1 所感・コメント

**v0.1**

```md
Reference Diff が一部で false なのは想定外。
```

**v0.2（書き換え例）**

```md
Reference Diff が false となっている AnswerPair が複数条例に分布している。
```

※ 評価語・感情語を除去し、
※ 「どこに現れているか」という事実関係のみを残す。

---

### 3.3 v0.1 結論・判断

**v0.1**

```md
今回は大きな問題はないと判断する。
```

**v0.2**

* 完全削除

※ 判断・結論は Evaluation Judgment フェーズ専用。

---

## 4. v0.2 HUMAN 記録で必ず残すもの

* run_id
* 参照している AUTO JSON のパス
* 注目した差分の種類（Reference / Volume / Structural）
* 判断を保留した理由
* 次に必要な比較・観点

---

## 5. 移行チェックリスト（実務用）

* [ ] AUTO JSON を参照している
* [ ] diff 件数・集計を書いていない
* [ ] 良否・評価語が含まれていない
* [ ] 判断を保留した理由が明示されている
* [ ] 次の Evaluation に接続できる観点がある

---

## 6. 移行後の位置づけ

v0.2 HUMAN 記録は以下のいずれでもない。

* Evaluation の結論
* Observation の再解釈

v0.2 HUMAN 記録は
**Evaluation Judgment を行うためのメタ情報ログ**である。
