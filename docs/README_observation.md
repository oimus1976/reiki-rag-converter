# Answer Diff Observation Artifact README

## 本成果物の目的

本成果物は、HTML 版 answer.md と Markdown 版 answer.md の差分を、
**評価・解釈・適否判断を行わず、観測事実として固定**したものです。

本成果物に含まれるデータは、Evaluation Framework における
評価・判断の **入力事実**として使用されることを想定しています。

---

## 成果物の構成

本ディレクトリには、以下のファイルが含まれます。

- `observation_result.json`  
  各 AnswerPair（条例 × 質問）ごとの観測結果を記録した JSON 正本です。  
  Reference Diff / Volume Diff / Structural Diff の結果が含まれます。

- `observation_summary.md`  
  観測仕様、Diff の定義、注意事項を記載した人間向けドキュメントです。  
  本成果物の解釈にあたっては、必ず本ファイルを参照してください。

- `README.md`（本ファイル）  
  本成果物セットの位置づけと取り扱い上の注意を記載します。

---

## 観測フェーズの責務範囲

本成果物は、**Observation フェーズの成果物**です。

- 差分の有無や量を、機械的・再現可能な方法で記録します
- 差分の良否・適切性・意味付けは行いません
- 観測結果の評価・解釈・判断は、本成果物の責務外です

評価・判断を行う場合は、Evaluation Framework 側で定義された
評価基準・文脈に従ってください。

---

## run_id について

本成果物は、特定の実行単位（run）に対応しています。

- run_id は、Observation 実行単位を識別するための識別子です
- 実行日時等に基づき一意に付与されます
- run_id 自体に、評価的・意味的な意味はありません

同一入力データであっても、別の run として実行された場合は、
異なる run_id を持つ成果物として扱われます。

---

## 注意事項

- 本成果物単体から、結論や優劣を導くことは意図されていません
- run_id の新旧や大小による優劣判断は行わないでください
- 観測結果の再解釈・再評価は、必ず Evaluation Framework を介してください
