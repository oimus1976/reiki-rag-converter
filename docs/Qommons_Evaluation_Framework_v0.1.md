# Qommons Evaluation Framework v0.1

本書は、Qommons.AI 上で
**Golden_Ordinance_Set に対して Golden_Question_Pool_A を実施した際の
テスト結果を評価するための公式評価軸**を定義する。

本評価は「品質スコアリング」ではなく、
**運用上の逸脱検知（Gate 判定）**を目的とする。

---

## 0. 前提（Scope & Assumptions）

本評価フレームワークは、以下を前提とする。

- ナレッジ構成は **Golden_Ordinance_Set のみ**であること
- Web検索は OFF であること
- 評価対象は **Qommons.AI の回答結果（Markdown / Text）**であること
- LLM の優劣比較・自然さ評価は行わない

---

### 0.1 Observation 成果物に関する前提

本評価フレームワークは、
Answer Diff Observation フェーズで生成された成果物を
**評価の前提入力**として利用する。

Observation 成果物は、以下の性質を持つものとする。

- Observation 成果物は、
  HTML 版 answer.md と Markdown 版 answer.md の差分を
  **評価や解釈を行わず、観測事実として固定**したものである
- 差分の有無・量・構造は、
  機械的かつ再現可能な方法で算出されている
- Observation 成果物には、
  差分の良否・適切性・意味付けは含まれない

本評価フレームワークでは、
Observation 成果物を **入力事実**として扱い、
その内容を再解釈・再計算・補正することは行わない。

評価・判断は、
本フレームワークで定義された評価軸に基づいて行うものとする。

---

## 1. 評価の基本思想

### 1.1 評価の目的

- Golden に対する **逸脱を検知する**
- 回帰テストとして **前回との差分を判断可能にする**
- 人手レビューを **最小限に抑え、体制として回る** ことを優先する

### 1.2 評価しないもの

以下は評価対象外とする。

- 表現の巧拙
- 文体の自然さ
- 回答の網羅性（聞かれていない情報）
- LLM 間の性能比較
- 点数化（○点 / △点 等）

---

## 2. 評価方式（Gate Model）

本評価は **Gate 方式**を採用する。

- 各評価軸は **OK / △ / NG** で判定する
- **NG が1つでもあれば「要調査」**
- 全軸 OK であれば「今回テストは合格」

---

## 3. 評価軸一覧（v0.1）

### 3.1 根拠準拠性（Citation Fidelity）

**定義**
回答が、実在する条例本文・条・項・附則にのみ基づいているかを評価する。

**チェック観点**
- 存在しない条文・制度を述べていないか
- 他条例・一般論・外部知識を混入させていないか
- 条文引用が誤っていないか

**判定**
- OK：すべて根拠あり
- NG：虚偽・混入あり（致命）

---

### 3.2 参照到達性（Retrieval Reach）

**定義**
質問で求められた条・項・附則に正しく到達できているかを評価する。

**チェック観点**
- 指定された条文に到達しているか
- 条ズレ（別条で回答）が起きていないか
- 附則を問われた際に本則で代替していないか

**判定**
- OK：正確に到達
- △：一部ズレ
- NG：到達失敗

---

### 3.3 構造保持性（Structural Integrity）

**定義**
条例の論理構造を崩さずに回答できているかを評価する。

**評価対象**
- 回答文中の構造（目的／手続／例外／附則 等）
- 表や別表の意味的構造（列・行の役割）

※ Qommons 内部のチャンク構造は評価対象外とする。

**チェック観点**
- 役割の混同がないか
- 表の意味を潰していないか

**判定**
- OK / NG

---

### 3.4 過剰補完耐性（Hallucination Control）

**定義**
条例に書かれていない事項について、適切に「ない」と扱えているかを評価する。

**チェック観点**
- 条例にない具体手続を勝手に補っていないか
- 一般論を断りなく条例の内容として述べていないか
- 条例外の説明には明示的な断りがあるか

**判定**
- OK
- NG（危険）

---

### 3.5 質問意図追従性（Intent Alignment）

**定義**
質問の意図に沿って回答できているかを評価する。

**チェック観点**
- 聞かれていない内容を過剰に説明していないか
- 要約・列挙・説明の粒度が大きくズレていないか

**判定**
- OK
- △（ズレあり）

---

## 4. Gate Question 運用

すべての質問を毎回評価する必要はない。

### 推奨 Gate Question（例）
- 義務・禁止抽出系（例：Q5）
- 定義条・用語系（例：Q11）
- 根拠提示強制（例：Q13）
- 統合説明（例：Q17）

Gate Question で NG が出た場合のみ、全体を再確認する。

---

## 5. 評価結果の扱い

- 評価結果は **合否・所感のみ** を記録する
- NG が出た場合は以下を切り分ける：
  - 入力（Golden_Ordinance_Set）の問題
  - 質問設計の問題
  - Qommons / LLM の劣化
- 本評価結果は Golden 更新判断の **補助資料** とする
