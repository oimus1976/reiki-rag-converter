## Qommons.AI 限界テスト標準手順書（v0.1）

本書は、Qommons.AI に条例データ（TXT／CSV／HTML）を登録し、
検索精度・統合回答・表処理能力・情報損失などの限界値を測定するための
**完全再現可能な標準テスト手順書**である。

得られた結果は以下に反映される：

- GOLDEN_POLICY_v1.0  
- convert_v2.8 / v3.0 の設計  
- TXT/CSV 出力方式の最適化  
- synthetic_html / validate の品質向上  

---

## 1. 目的

Qommons.AI の検索挙動を体系的に評価し、  
**例規データの最適フォーマット（GOLDEN形式）を確立する根拠データ**を収集する。

対象となる評価ポイント：

- TXT / CSV の統合回答能力  
- HTMLタグ削除後の情報損失  
- モデルごとの検索精度差  
- チャンク化（文字数ベース）の影響  
- テーブル認識精度（colspan/rowspan含む）  
- ノイズ混在時のスコア低下

---

## 2. テスト対象

### 2.1 ファイル形式

- TXT：条例本文（Markdown互換テキスト）
- CSV：別表（説明行つきヘッダー必須）
- HTML：自治体公式例規HTML（タグ削除前の生データ）

### 2.2 モデル（標準評価モデル）

| 用途 | モデル |
|------|--------|
| 基準評価 | Claude 4.5 Sonnet（国内リージョン） |
| 汎用比較 | ChatGPT 5.1 |
| 表理解検証 | Gemini 2.5 Flash |
| 国内制約確認 | PLaMo 2.1 Prime |

---

## 3. 事前設定（必須）

### 3.1 Web検索
- **OFF（必須）**

### 3.2 ナレッジ公開範囲
- **所属部署**（共有はノイズ混入の可能性あり）

### 3.3 ナレッジ選択
- Qommons の UI 制約上、  
  **横断検索を行う場合は「所属部署」ナレッジに複数ファイルを登録して使う。**

### 3.4 チャット開始時の初期メッセージ

```planetext
これから Qommons テストを行います。
参照するのはナレッジ内の条例ファイルのみです。
```

---

## 4. ファイルアップロード手順

#### (1) TXT + CSV の基本テスト

1. 所属部署ナレッジを開く
2. ファイルを次の順でアップロード  
   **TXT → CSV**  
3. この順序でチャンクが安定する

#### (2) HTML 単体テスト

1. HTML のみをアップロード  
2. TXT/CSV を混ぜない  
3. タグ削除 → 文字列化後の情報喪失を確認

#### (3) HTML + TXT/CSV 混在

1. **TXT → CSV → HTML** の順  
2. HTML がノイズ源として影響するかを評価

---

## 5. ナレッジ選択ルール

- 「単一ファイル選択モード」では **横断回答不可**  
- CSV の説明行・語彙統一が検索精度に影響  
- ノイズ混在テストでは、ダミー条例を 10〜20 個混在させる

---

## 6. 質問前の準備

1. 新しいチャットに切り替える  
2. 初期メッセージ送信  
3. 評価対象モデルに変更  
4. Web検索OFFを確認

---

## 7. テストケース実行手順（質問セットの運用方法）

1. 質問は **1件ずつ**  
2. 回答に引用元が含まれるか必ず確認  
3. 要約で終わる場合は「根拠を提示して」と再質問  
4. 表を含む質問では CSV/TXT の両方を参照するか確認  
5. HTML混在テストでは“誤引用”を重点評価

---

## 8. 成功基準・失敗基準

### 成功（Pass）

- 正答または部分正答率 80%以上  
- TXT + CSV の横断回答成功率 70%以上  
- 表の行列が保持されている  
- 回答に引用元（TXT/CSV/HTML）が明示される  

### 失敗（Fail）

- HTML 混在で TXT/CSV が参照されない  
- 表の構造（行/列）が失われる  
- 誤引用が 30% 以上  
- モデル間の回答差異が極端に大きい

---

## 9. ログ保存方法

- スクリーンショットを取得  
- 回答全文を Markdown 化して保存  
- 保存先： `test_results/Qommons/YYYYMMDD/条例ID_モデル名.md`
